Что понадобится

VDS/VPS с Ubuntu 22.04 или 24.04, доступ по SSH. Порт 443 должен быть свободен. Docker установим в процессе.

Шаг 1

Создаём облачный сервер

Если сервер уже есть — переходите к следующему шагу. Если нет — создаём новый.

Прямая ссылка для создания

timeweb.cloud/my/servers/create

Откроется форма с предвыбранной конфигурацией.

📋 Настройки при создании

Операционная система: Ubuntu 24.04 LTS
Локация: Нидерланды (Amsterdam) или Казахстан
Тариф: минимальный (1 CPU, 1 GB RAM достаточно)
Дополнения: все отключить (бэкапы, мониторинг и т.д.)

⚠️ Важно по локации

Выбирайте Нидерланды или Казахстан — оттуда стабильный канал до серверов Telegram. Российские локации могут давать блокировки.

Шаг 2

Находим данные для входа

После создания сервера заходим в его дашборд — там есть всё необходимое для подключения.

Дашборд сервера → IPv4-адрес, SSH-команда и Root-пароль

Что нам нужно

IPv4 — это ваш ВАШ_IP, который используется далее
Подключение по SSH — готовая команда, скопируйте одним кликом
Root-пароль — нажмите на иконку копирования рядом со звёздочками

Шаг 3

Подключаемся по SSH

Открываем терминал (Terminal на macOS/Linux, PowerShell или PuTTY на Windows) и вводим команду.

terminal 
ssh root@ВАШ_IP

⚠️ Первое подключение

Терминал спросит подтверждение отпечатка — введите yes. Затем вставьте пароль из панели (он не отображается при вводе — это нормально).

Так выглядит успешное подключение — приветственное сообщение Ubuntu

Шаг 4

Устанавливаем Docker

Если Docker уже установлен — проверьте командой docker -v. Если нет — ставим официальным скриптом.

Обновление пакетов 
apt-get update -y && apt-get upgrade -y
Установка Docker 
curl -fsSL https://get.docker.com | sh
systemctl enable docker && systemctl start docker
Проверка 
docker -v
# Ожидаем: Docker version 2x.x.x ...
Шаг 5

Проверяем порт 443

MTProxy с Fake TLS должен работать на порту 443, чтобы маскироваться под HTTPS. Убедимся, что он свободен.

Проверка занятых портов 
ss -tulpn | grep :443

✅ Порт свободен

Если команда ничего не вывела — отлично, переходим к следующему шагу.

⚠️ Порт занят (nginx, apache...)

Остановите мешающий сервис или выберите другой порт (например, 8443). В дальнейших командах замените 443 на выбранный порт.

Остановить nginx / apache (если нужно) 
systemctl stop nginx 2>/dev/null
systemctl stop apache2 2>/dev/null
Шаг 6

Генерируем секрет (Fake TLS)

Секрет с префиксом ee активирует Fake TLS — трафик маскируется под обращение к Google.

Генерация секрета 
docker run --rm nineseconds/mtg:2   generate-secret --hex www.google.com

📋 Сохраните вывод

Команда выведет строку вида ee_Ваш_Секрет. Скопируйте её целиком — она нужна в следующем шаге.

Почему именно Google?

Домен-маскировка должен быть популярным и стабильным HTTPS-сайтом, доступным в РФ. YouTube и Cloudflare частично блокируются — используем www.google.com.

Шаг 7

Запускаем прокси

Подставьте секрет из предыдущего шага вместо ee_Ваш_Секрет и запустите контейнер.

Запуск MTProxy 
docker run -d \
  --name mtproto-proxy \
  --restart unless-stopped \
  -p 443:443 \
  nineseconds/mtg:2 \
  simple-run -n 1.1.1.1 -i prefer-ipv4 \
  0.0.0.0:443 ee_Ваш_Секрет

Что означают флаги

--restart unless-stopped — автозапуск при перезагрузке сервера
-n 1.1.1.1 — DNS-резолвер Cloudflare
-i prefer-ipv4 — предпочитать IPv4 соединения

Шаг 8

Проверяем работу

Убедимся, что контейнер запустился и порт 443 прослушивается.

Статус контейнера 
docker ps | grep mtproto
# Ожидаем: Up X seconds   0.0.0.0:443->443/tcp
Порт и логи 
ss -tulpn | grep :443
docker logs mtproto-proxy

✅ Всё в порядке, если

Статус контейнера Up, порт 443 присутствует в выводе ss, в логах нет строк ERROR.

Шаг 9

Открываем порт в файрволе

Рекомендуем через консоль — это быстрее. Альтернативно — через панель Timeweb Cloud.

Способ 1 — Консоль (рекомендуется) 
ufw allow 443/tcp
ufw status
# Если ufw inactive — файрвол не мешает, шаг можно пропустить

Способ 2 — Панель Timeweb Cloud

Ваш сервер → вкладка Сеть → блок Firewall → кнопка Настроить. Добавьте правило: TCP, порт 443, источник 0.0.0.0/0.

Вкладка «Сеть» → блок Firewall → кнопка «Настроить»

Шаг 10

Подключаем в Telegram

Два способа: по ссылке (быстро) или вручную через настройки.

Способ 1 — Магическая ссылка

Сформируйте ссылку ниже, отправьте себе в «Избранное» и тапните по ней:

tg://proxy?server=ВАШ_IP&port=443&secret=ee_Ваш_Секрет

Способ 2 — Вручную

Настройки → Данные и память → Прокси → Добавить прокси

Тип: MTProto
Сервер: ВАШ_IP
Порт: 443
Секрет: ee_Ваш_Секрет

Готово!

Прокси работает 🎉

После подключения в Telegram появится иконка щита. Медиа загружаются через ваш сервер.

✅ Признаки успешной работы

Иконка щита в верхней панели Telegram, статус «Подключено» в настройках прокси, ускорение загрузки медиа.

Управление контейнером 
# Перезапустить
docker restart mtproto-proxy

# Активные подключения
docker exec mtproto-proxy ss -tn 2>/dev/null | grep :443 | wc -l

# Логи в реальном времени
docker logs -f mtproto-proxy
